# Contributing to Awesome Rust CLI Tools

Thank you for your interest in contributing! This list aims to be a high-quality, curated resource of Rust CLI tools with at least 10 GitHub stars.

## How to Contribute

1. **Fork this repository**
2. **Create a new branch** for your changes
3. **Edit the README.md** to add your tool
4. **Ensure your entry follows the format**:
   ```
   | [tool-name](https://github.com/user/repo) | Brief, benefit-focused description (15-25 words) | ⭐ X,XXX | YYYY-MM-DD |
   ```
5. **Verify the tool meets criteria**:
   - Written primarily in Rust
   - Has at least 10 stars on GitHub
   - Is a CLI tool (not just a library)
   - Last updated within the last 6 months (preferred)
   - Provides genuine utility as a command-line utility
6. **Submit a Pull Request** with a clear description

## Adding a New Tool

When adding a tool, please:

- Place it in the appropriate category
- Keep alphabetical ordering within each category
- Use the exact format: `[tool-name](https://github.com/user/repo)` - Description
- Keep descriptions concise and benefit-focused
- Include star count using the format: `⭐ 1,234`
- Include date of last significant update (YYYY-MM-DD)
- Do not include personal opinions or markdown formatting in descriptions

## Removing Tools

If you notice a tool that no longer meets the criteria (abandoned, dropped below 10 stars, etc.), please open an issue or submit a PR to remove it.

## Questions?

Feel free to open an issue if you have questions about contributing.

## How to Request Your Tool Be Added

If you have a Rust-based command-line tool you'd like to add to this list, please open an issue with the following information:

- **Tool name**: The name of your tool
- **GitHub repository**: Full URL to the repository
- **Description**: A brief, benefit-focused description (15-25 words)
- **Stars**: Current star count (must be ≥10)
- **Why it belongs**: Which category it fits into and why it's a good addition

After your issue is reviewed and accepted, you can create a pull request with the proper format.

### Pull Request Format

Please follow this format when adding your tool:

```markdown
| [tool-name](https://github.com/user/repo) | Brief, benefit-focused description highlighting what problem it solves | ⭐ 1,234 | 2026-01-01 |
```

Make sure to:
- Place it in the appropriate category section
- Keep alphabetical ordering within the category
- Verify the tool meets all criteria listed above
- Keep descriptions concise and focused on user benefits
